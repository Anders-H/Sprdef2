About Sprdef2

This project started with a simple belief: Developing software for the Commodore 64 should be a smooth, efficient, and thoroughly enjoyable experience. As a retro developer myself, I noticed a gap in the workflow when it came to sprite design. I wanted a modern Windows tool that didn't get in the way of creativity — a tool that does the right things exceptionally well.

Sprdef2 was built to fix that. By focusing on seamless integration with CBM Prg Studio and enabling direct data export to the C64, it removes the tedious bottlenecks from your pipeline so you can focus on what actually matters: making great games. This editor is my contribution to the amazing retro computing community that keeps the C64 alive. It’s free, and it is open source!

License Apache 2.0

Core Values

  • Workflow-Driven Integration: We believe tools should eliminate friction, not create it. Sprdef2 is designed to adapt to how modern retro-developers actually work. By integrating seamlessly with essential software like CBM Prg Studio and offering direct export to the Commodore 64, it bridges the gap between modern PC convenience and classic hardware. It’s not just about editing sprites; it’s about perfecting the pipeline.
  • Purpose-Built Simplicity: Software should do the right things, exceptionally well. We don’t believe in bloatware or over-complicating the creative process. Sprdef2 focuses on delivering the exact features a C64 developer needs, without the clutter. Every function is intentional, intuitive, and built with a clear purpose: to make sprite design clean, fast, and efficient.
  • Keeping the Joy of Retro Alive: Developing for classic hardware should be as fun as playing the games. Passion is at the heart of homebrew development. We build tools that protect and enhance that passion. By removing tedious technical hurdles and streamlining the design process, Sprdef2 keeps the workflow engaging, letting developers focus on the pure joy of creating for the Commodore 64.
